### 2. Right to Restriction (GDPR Art. 18)
```
CURRENT STATUS: ❌ Not built

WHAT IS NEEDED:
  → User can request to RESTRICT processing
    of their data (not delete, just pause)
  → Different from right to erasure
  → Must be honoured within 1 month

WHAT TO BUILD:
  → "Restrict my data processing" request
    in GDPR Rights page
  → Flag on user account: processing_restricted
  → When restricted: account becomes read-only
    no marketing, no profiling, no exports
  → Admin sees restriction flag on user profile

EFFORT: Low (1–2 days)
```

### 3. Data Breach — Automated Detection Hints
```
CURRENT STATUS: Manual logging only

WHAT IS MISSING:
  → Currently super admin manually logs breach
  → No automated suspicious activity detection
    that HINTS a breach may have occurred

WHAT TO BUILD:
  → Automated flags in security alerts:
    "Unusual: 500 failed logins in 10 min"
    "Unusual: bulk data export at 3 AM"
    "Unusual: same IP accessing 50 accounts"
  → These are HINTS — not breach declarations
  → Super admin investigates and decides
    whether to log as formal breach

EFFORT: Medium (3–5 days)
```

### 4. Cookie Consent — Granular Categories
```
CURRENT STATUS: Basic banner built

WHAT IS MISSING:
  → Necessary / Analytics / Marketing
    categories need to actually
    BLOCK/ALLOW scripts based on selection
  → Currently records preference but does
    not enforce it technically

WHAT TO BUILD:
  → If Analytics rejected:
    → Google Analytics / tracking blocked
  → If Marketing rejected:
    → Meta pixel, LinkedIn pixel blocked
  → Preference respected on every page load

EFFORT: Low (1 day)
  BUT: only matters when you add
  analytics/marketing scripts
  Not urgent until then
```
