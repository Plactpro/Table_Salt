## PROMPT 1 – Kitchen Display System (Chef-Facing UI)

You are rebuilding the **Kitchen Display System (KDS)** for a multi-tenant restaurant SaaS. The UI must work across **3 hardware scenarios** without code changes — just configuration per outlet:

- **Scenario A**: Wall-mounted kitchen screen only (no chef tablets)
- **Scenario B**: Chef tablets only (no wall screen)
- **Scenario C**: Both wall screen + chef tablets simultaneously

The system must **auto-detect or be configured** per outlet which scenario applies and render the correct UI automatically.

***

### Display mode configuration (per outlet setting)

```sql
CREATE TABLE outlet_display_config (
  id              UUID PRIMARY KEY,
  tenant_id       UUID NOT NULL,
  outlet_id       UUID NOT NULL,
  display_mode    VARCHAR(20) NOT NULL,
    -- 'WALL_SCREEN', 'TABLET', 'BOTH'
  wall_screen_url VARCHAR(255),
  auto_assign     BOOLEAN DEFAULT false,
  created_at      TIMESTAMP DEFAULT NOW()
);
```

- Manager sets display mode in outlet settings.
- `WALL_SCREEN` mode: full-screen, no login required, auto-refresh, large fonts.
- `TABLET` mode: chef logs in, sees only their station tickets.
- `BOTH` mode: wall screen shows all tickets, tablets show per-chef/station.

## PROMPT 2 – Auto-Send to Kitchen + Stock Deduction Engine

You are implementing the **order-to-kitchen auto-routing and stock deduction engine** for a multi-tenant restaurant SaaS.

***

### Auto-send to kitchen on order creation

- When a waiter submits an order from POS:
  - Immediately and automatically route to kitchen — **no manual "send to kitchen" button required**.
  - Split the order into **KOT lines per station** based on menu item → station mapping:
    ```
    Order #045 (Table 5):
    → GRILL station:   Grilled Chicken x1, Steak x2
    → BAR station:     Coke x2, Fresh Lime x1
    → DESSERT station: Chocolate Cake x1
    ```
  - Each station receives only its relevant items.
  - Broadcast to wall screen and relevant chef tablets via WebSocket instantly.
- Log the KOT event:
  ```sql
  CREATE TABLE kot_events (
    id           UUID PRIMARY KEY,
    tenant_id    UUID NOT NULL,
    outlet_id    UUID NOT NULL,
    order_id     UUID NOT NULL,
    station      VARCHAR(100),
    items        JSONB NOT NULL,
    sent_at      TIMESTAMP DEFAULT NOW(),
    received_at  TIMESTAMP
  );
  ```

***

### Stock deduction engine (when chef confirms preparation)

- Trigger: chef taps "Confirm & Start" on tablet OR manager clicks "Start" on wall screen.
- For each menu item being prepared:
  - Look up linked recipe and components.
  - Calculate required quantities:
    ```
    required_qty = component.quantity × (portions / recipe.yield)
    convert to base unit using conversion_factor
    ```
  - Check stock availability **before deducting**:
    - If sufficient: deduct and log.
    - If insufficient: show alert (see PROMPT 1) but allow override with reason.
  - Deduct from `ingredients.current_stock`.
  - Log full transaction:

```sql
INSERT INTO inventory_transactions (
  tenant_id, outlet_id, ingredient_id, ingredient_name,
  transaction_type,   -- 'RECIPE_CONSUMPTION'
  quantity_change,    -- negative
  stock_before, stock_after,
  order_id, order_number,
  menu_item_id, menu_item_name,
  recipe_id,
  chef_id, chef_name, station,
  shift_id,
  created_at
) VALUES (...);
```

- Wrap ALL deductions for one order in a **single DB transaction** (atomic — all succeed or all rollback).
- On order void / cancel / refund:
  - Reverse all deductions with `type = 'RECIPE_REVERSAL'` and positive quantities.

***

### Menu item → station mapping

```sql
CREATE TABLE menu_item_stations (
  id           UUID PRIMARY KEY,
  tenant_id    UUID NOT NULL,
  menu_item_id UUID NOT NULL,
  station      VARCHAR(100) NOT NULL
    -- 'GRILL', 'BAR', 'COLD_KITCHEN',
    -- 'DESSERT', 'FRYER', 'MAIN_KITCHEN'
);
```

- Manager assigns each menu item to a station in Menu Settings.
- This drives automatic KOT routing and chef tablet filtering.
