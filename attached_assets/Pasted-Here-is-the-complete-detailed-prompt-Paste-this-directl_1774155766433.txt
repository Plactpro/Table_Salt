### Prompt for Replit AI – Automated Daily Stock Check & Menu Capacity Report

This is a continuation of the existing restaurant SaaS project. The following modules are already built and working: multi-tenant architecture, inventory management, recipe-to-inventory linking, menu management, chef assignment, and shift roster.

Now build a **fully automated Daily Stock Check and Menu Capacity Report system** that runs automatically before every day or shift, analyzes current stock against all recipes, and generates a detailed report showing exactly how many portions of each dish can be prepared.

***

#### 1. Core concept and flow

```
AUTOMATED TRIGGER (scheduled job or manual)
        ↓
Pull all ACTIVE MENU ITEMS for tomorrow / next shift
        ↓
For each menu item → fetch linked RECIPE + COMPONENTS
        ↓
Check current STOCK LEVEL of each ingredient
        ↓
Calculate MAX PORTIONS possible per dish
        ↓
Identify BOTTLENECK INGREDIENTS (limiting factors)
        ↓
Flag CRITICAL SHORTAGES (cannot prepare even 1 portion)
        ↓
Generate DETAILED REPORT
        ↓
Auto-send to Manager + Owner (email + in-app notification)
        ↓
Manager can ACTION from report:
  ├── Raise Purchase Order
  ├── Disable dish for tomorrow
  └── Adjust planned quantity
```

#### 3. Core calculation engine

Implement `StockCapacityService` with this logic:

```typescript
interface DishCapacity {
  menuItemId: string;
  menuItemName: string;
  category: string;
  recipeId: string;
  maxPortions: number;           // max dishes can be made with current stock
  bottleneckIngredient: string;  // ingredient limiting the portions
  bottleneckStock: number;       // current stock of bottleneck ingredient
  bottleneckRequired: number;    // required per portion
  status: 'SUFFICIENT'           // can make planned qty
         | 'LIMITED'             // can make some but not full planned qty
         | 'CRITICAL'            // can make < 20% of planned qty
         | 'UNAVAILABLE';        // cannot make even 1 portion
  ingredientBreakdown: IngredientStatus[];
}

interface IngredientStatus {
  ingredientId: string;
  ingredientName: string;
  requiredPerPortion: number;
  unit: string;
  currentStock: number;
  maxPortionsThisIngredient: number;  // stock / required_per_portion
  stockStatus: 'OK' | 'LOW' | 'CRITICAL' | 'OUT_OF_STOCK';
  shortfallForPlannedQty: number;     // how much more needed
  estimatedCost: number;              // cost to restock shortfall
}

async function calculateMenuCapacity(
  outletId: string,
  targetDate: Date,
  plannedQuantities?: Record<string, number>  // optional planned qty per dish
): Promise<DishCapacity[]> {

  // 1. Get all active menu items for target date
  const menuItems = await getActiveMenuItems(outletId, targetDate);

  // 2. For each item, calculate max portions
  const capacities = await Promise.all(
    menuItems.map(async (item) => {

      // Get recipe and components
      const recipe = await getRecipeWithComponents(item.recipeId);
      if (!recipe) return buildNoRecipeResult(item);

      // For each ingredient, calculate max portions possible
      const ingredientStatuses = await Promise.all(
        recipe.components.map(async (component) => {
          const stock = await getCurrentStock(
            component.ingredientId,
            outletId
          );
          const requiredPerPortion =
            component.quantity / recipe.yieldQuantity;
          const maxFromThisIngredient =
            Math.floor(stock / requiredPerPortion);

          return {
            ...component,
            currentStock: stock,
            requiredPerPortion,
            maxPortionsThisIngredient: maxFromThisIngredient,
            stockStatus: getStockStatus(stock, requiredPerPortion),
            shortfallForPlannedQty: calculateShortfall(
              stock,
              requiredPerPortion,
              plannedQuantities?.[item.id] ?? defaultPlannedQty
            )
          };
        })
      );

      // Max portions = minimum across all ingredients (bottleneck)
      const maxPortions = Math.min(
        ...ingredientStatuses.map(i => i.maxPortionsThisIngredient)
      );
      const bottleneck = ingredientStatuses.find(
        i => i.maxPortionsThisIngredient === maxPortions
      );

      return {
        menuItemId: item.id,
        menuItemName: item.name,
        category: item.category,
        maxPortions,
        bottleneckIngredient: bottleneck?.ingredientName,
        status: getCapacityStatus(maxPortions, plannedQuantities?.[item.id]),
        ingredientBreakdown: ingredientStatuses
      };
    })
  );

  return capacities.sort((a, b) => a.maxPortions - b.maxPortions);
}
```

#### 6. Action buttons from report

Each dish in the report has direct action buttons:

##### Raise Purchase Order
- One-click from report to create a PO for the shortfall ingredient.
- Pre-fills:
  - Ingredient name.
  - Required quantity (shortfall amount).
  - Preferred supplier (from ingredient master).
  - Urgency: HIGH (needed by tomorrow).
- Manager can review and approve PO from the same report screen.

##### Disable Dish for Tomorrow
- One-click to mark the dish as **unavailable** for the target date.
- Automatically:
  - Hides dish on QR menu for tomorrow.
  - Flags dish as unavailable on POS.
  - Shows "Not Available Today" badge on menu.
- Manager can undo this action.

##### Set Maximum Quantity
- Manager enters a max serve limit for tomorrow (e.g., "Only serve 12 Soups").
- System shows "Limited Availability" on QR menu after X portions sold.
- POS shows remaining count next to dish name.

##### Bulk actions (from report header)
- **Raise All POs**: creates purchase orders for all critical/unavailable items in one click.
- **Disable All Critical**: disables all dishes with CRITICAL or UNAVAILABLE status.
- **Export PDF**: full report as print-ready PDF for head chef morning briefing.
- **Export CSV**: full data export for further analysis.

#### 9. Planned quantity management

Allow manager to set **planned quantities per dish per day**:

```sql
CREATE TABLE daily_planned_quantities (
  id              UUID PRIMARY KEY,
  tenant_id       UUID NOT NULL,
  outlet_id       UUID NOT NULL,
  menu_item_id    UUID NOT NULL,
  planned_date    DATE NOT NULL,
  planned_qty     INT NOT NULL,
  actual_qty_sold INT DEFAULT 0,
  max_limit       INT,
  is_disabled     BOOLEAN DEFAULT false,
  disabled_reason TEXT,
  created_by      UUID,
  updated_at      TIMESTAMP DEFAULT NOW(),
  UNIQUE(tenant_id, outlet_id, menu_item_id, planned_date)
);
```

- Manager can set planned quantities in two ways:
  - **Daily planner** screen: set qty per dish for any future date.
  - **From the stock report**: adjust planned qty directly in the report.
- If no planned quantity set: system uses **default average** from last 7 days sales data.
