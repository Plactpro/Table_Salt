### Order Ticket History — Phase 1 (View, Reprint, Void, Refire)

This is a continuation of the existing restaurant SaaS project in Replit. The following are already built: multi-tenant architecture, POS, KDS, order management, billing, printer integration, service coordination, and time tracking.

Now build **Phase 1 of the Order Ticket History system** — focused only on the 4 features used daily: view old tickets, reprint documents, void items, and refire items to kitchen. Keep everything fast, simple, and practical.

***

#### 1. What to build in Phase 1 only

```
✅ FEATURE 1: Ticket history search and view
   → Search any past order instantly
   → See full details including modifications,
     timings, payment, and staff

✅ FEATURE 2: Reprint (receipt, KOT, bill)
   → One-tap reprint from any past ticket
   → Logged automatically

✅ FEATURE 3: Void item with approval
   → Void wrong/cancelled item on active
     or recently closed order
   → Manager approval required
   → Audit log written automatically

✅ FEATURE 4: Refire item to kitchen
   → Send a specific item back to kitchen
     for remake
   → New KOT generated with REFIRE header
   → Linked to original order

❌ NOT IN PHASE 1:
   Full order reopen, payment change,
   add item after close, partial refund,
   reopen session timer, split bill correction
```

#### 2. Database schema — Phase 1 only

```sql
-- Audit trail for every action
CREATE TABLE order_audit_trail (
  id                  UUID PRIMARY KEY,
  tenant_id           UUID NOT NULL,
  order_id            UUID NOT NULL,
  order_number        VARCHAR(50),
  action_type         VARCHAR(50) NOT NULL,
    -- 'TICKET_VIEWED', 'RECEIPT_REPRINTED',
    -- 'KOT_REPRINTED', 'BILL_REPRINTED',
    -- 'ITEM_VOID_REQUESTED', 'ITEM_VOID_APPROVED',
    -- 'ITEM_VOID_REJECTED', 'ITEM_VOIDED',
    -- 'ITEM_REFIRE_REQUESTED', 'ITEM_REFIRED'
  action_description  TEXT NOT NULL,
  previous_value      JSONB,
  new_value           JSONB,
  performed_by        UUID NOT NULL,
  performed_by_name   VARCHAR(255),
  performed_by_role   VARCHAR(50),
  created_at          TIMESTAMP DEFAULT NOW()
);

-- Void requests
CREATE TABLE item_void_requests (
  id                  UUID PRIMARY KEY,
  tenant_id           UUID NOT NULL,
  outlet_id           UUID NOT NULL,
  order_id            UUID NOT NULL,
  order_number        VARCHAR(50),
  order_item_id       UUID NOT NULL,
  menu_item_name      VARCHAR(255),
  quantity            INT,
  unit_price          DECIMAL(10,2),
  total_value         DECIMAL(10,2),
  void_reason         TEXT NOT NULL,
  void_type           VARCHAR(30) NOT NULL,
    -- 'WRONG_ORDER', 'CUSTOMER_CANCELLED',
    -- 'KITCHEN_ERROR', 'QUALITY_ISSUE',
    -- 'DUPLICATE_ENTRY', 'MANAGER_VOID'
  status              VARCHAR(20) DEFAULT 'pending',
    -- 'pending', 'approved', 'rejected'
  requested_by        UUID NOT NULL,
  requested_by_name   VARCHAR(255),
  requested_by_role   VARCHAR(50),
  approved_by         UUID,
  approved_by_name    VARCHAR(255),
  rejected_reason     TEXT,
  approved_at         TIMESTAMP,
  created_at          TIMESTAMP DEFAULT NOW()
);

-- Voided items record
CREATE TABLE voided_items (
  id                  UUID PRIMARY KEY,
  tenant_id           UUID NOT NULL,
  order_id            UUID NOT NULL,
  order_item_id       UUID NOT NULL,
  void_request_id     UUID NOT NULL,
  menu_item_name      VARCHAR(255),
  quantity            INT,
  unit_price          DECIMAL(10,2),
  total_value         DECIMAL(10,2),
  void_reason         TEXT,
  void_type           VARCHAR(30),
  voided_by           UUID NOT NULL,
  voided_by_name      VARCHAR(255),
  approved_by         UUID NOT NULL,
  approved_by_name    VARCHAR(255),
  created_at          TIMESTAMP DEFAULT NOW()
);

-- Refire requests
CREATE TABLE item_refire_requests (
  id                  UUID PRIMARY KEY,
  tenant_id           UUID NOT NULL,
  outlet_id           UUID NOT NULL,
  order_id            UUID NOT NULL,
  order_number        VARCHAR(50),
  order_item_id       UUID NOT NULL,
  menu_item_name      VARCHAR(255),
  quantity            INT,
  refire_reason       TEXT NOT NULL,
    -- 'CUSTOMER_RETURNED', 'WRONG_PREPARATION',
    -- 'MODIFICATION_MISSED', 'DAMAGED_IN_SERVICE',
    -- 'QUALITY_ISSUE'
  priority            VARCHAR(20) DEFAULT 'high',
    -- always high since customer is waiting
  assign_to_chef_id   UUID,
  assign_to_chef_name VARCHAR(255),
  new_kot_number      VARCHAR(50),
  status              VARCHAR(20) DEFAULT 'sent',
    -- 'sent', 'acknowledged', 'cooking', 'ready'
  requested_by        UUID NOT NULL,
  requested_by_name   VARCHAR(255),
  created_at          TIMESTAMP DEFAULT NOW()
);
```

#### 9. APIs required — Phase 1 only

```
-- Ticket history
GET    /api/tickets/history              → list with filters
GET    /api/tickets/:orderId             → full ticket detail
GET    /api/tickets/:orderId/timeline    → audit trail

-- Reprint
POST   /api/tickets/:orderId/reprint/receipt  → reprint receipt
POST   /api/tickets/:orderId/reprint/kot      → reprint KOT
POST   /api/tickets/:orderId/reprint/bill     → reprint bill
POST   /api/tickets/:orderId/send-whatsapp    → send receipt
POST   /api/tickets/:orderId/send-email       → send receipt

-- Void
POST   /api/tickets/:orderId/void-request     → request void
GET    /api/tickets/void-requests/pending     → pending approvals
PUT    /api/tickets/void-requests/:id/approve → approve void
PUT    /api/tickets/void-requests/:id/reject  → reject void

-- Refire
POST   /api/tickets/:orderId/refire           → send item to kitchen
GET    /api/tickets/:orderId/refires          → refire history
```
