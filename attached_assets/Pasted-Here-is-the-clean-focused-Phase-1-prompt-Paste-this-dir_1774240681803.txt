### Prompt for Replit AI – Food Modification System (Phase 1)

This is a continuation of the existing restaurant SaaS project. POS, QR ordering, KDS, chef assignment, and order management are already built and working.

Now add **Phase 1 food modification support** to every order item. Keep it simple, fast, and practical — covering the 5 core modifications that handle 90% of real restaurant requests. Maximum 2 taps to add any modification. Must work seamlessly on POS (waiter), QR (customer), and KDS (chef).

#### 2. Where modifications appear

##### In POS (waiter entry)
- After adding any item to cart, a **"Customize"** button appears on the cart line.
- Tapping it opens a **bottom drawer** with all 5 modification sections.
- Sections are compact and collapsible except Allergy (always expanded).
- Waiter can complete in under 10 seconds.
- Modified items show a small **"✏️ customized"** tag on cart line.

##### In QR ordering (customer)
- After tapping any menu item, show item detail page.
- Modifications appear below item description in collapsible sections.
- Allergy section always visible and never collapsible.
- Customer sees live price update if any paid extras added (Phase 2).
- Summary of selected modifications shown as chips before "Add to Order".

##### On KDS ticket (chef)
- Modifications listed clearly below each item name.
- Color-coded per type:
  - 🌶️ Spice level — orange text
  - 🧂 Salt level — blue text
  - ➖ Removals — red strikethrough style
  - ✏️ Special note — yellow highlighted text
  - 🚨 Allergy — red bordered box, flashing border
- Allergy box has an **"I UNDERSTAND ✓"** button chef must tap.
- Items with modifications have a small **"MOD"** badge on ticket card.

##### On printed/digital receipt
```
Butter Chicken                      ₹380
  Spice: Mild
  Salt: Less Salt
  Removed: Onion, Cream
  ⚠️ Allergy: Dairy — Chef Verified
  Note: "Extra sauce on the side"
```

#### 3. Database schema (minimal and clean)

```sql
-- Stores modifications per order item
CREATE TABLE order_item_modifications (
  id                UUID PRIMARY KEY,
  tenant_id         UUID NOT NULL,
  order_id          UUID NOT NULL,
  order_item_id     UUID NOT NULL,
  spice_level       VARCHAR(20),
    -- 'NO_SPICE', 'MILD', 'MEDIUM', 'SPICY', 'EXTRA_HOT'
    -- NULL means not specified (use default)
  salt_level        VARCHAR(20),
    -- 'LESS', 'NORMAL', 'EXTRA'
    -- NULL means not specified (use default)
  removed_ingredients JSONB DEFAULT '[]',
    -- ["Onion", "Cream", "Garlic"]
  allergy_flags     JSONB DEFAULT '[]',
    -- ["Dairy", "Nuts", "Other: Tree nuts"]
  has_allergy       BOOLEAN DEFAULT false,
    -- quick filter flag
  allergy_severity  VARCHAR(20),
    -- 'life_threatening', 'intolerance', 'preference'
  special_note      VARCHAR(150),
  chef_acknowledged BOOLEAN DEFAULT false,
  chef_id           UUID,
  acknowledged_at   TIMESTAMP,
  created_at        TIMESTAMP DEFAULT NOW(),
  updated_at        TIMESTAMP DEFAULT NOW()
);

-- Flag removable ingredients per recipe component
ALTER TABLE recipe_components
  ADD COLUMN IF NOT EXISTS is_removable BOOLEAN DEFAULT true;
  -- false = core ingredient, cannot be removed (e.g. rice in biryani)
```

#### 4. Allergy alert flow (WebSocket events)

```typescript
// On order item with allergy flag submitted:

// 1. Emit to chef assigned to this item
socket.to(`chef_${chefId}`).emit('allergy_alert', {
  orderId,
  orderNumber,
  tableNumber,
  itemName,
  allergyFlags,    // ["Dairy", "Nuts"]
  severity,
  message: `🚨 ALLERGY: Table ${tableNumber} — ${itemName} has ${allergyFlags.join(', ')} allergy`
});

// 2. Emit to head chef / coordinator
socket.to(`outlet_${outletId}_managers`).emit('allergy_alert', {
  orderId,
  tableNumber,
  itemName,
  allergyFlags,
  severity
});

// 3. Chef acknowledgement
socket.on('allergy_acknowledged', async ({ modificationId, chefId }) => {
  await markAllergyAcknowledged(modificationId, chefId);
  socket.to(`order_${orderId}`).emit('allergy_confirmed', {
    itemName,
    chefName,
    confirmedAt: new Date()
  });
});
```

#### 6. Validation rules

```
- Spice and salt: single selection only, deselectable
- Remove ingredients: max 5 removals per item
- Allergy flags: multiple allowed, no limit
- Special note: max 150 characters, trim whitespace
- If allergy flagged: chef acknowledgement is MANDATORY
  before item can be marked "in preparation"
- If recipe contains flagged allergen:
  show warning to waiter/customer but do not block order
  (customer/waiter confirms they want to proceed)
```
