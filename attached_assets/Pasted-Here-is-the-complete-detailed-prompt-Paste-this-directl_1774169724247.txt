#### 4. Order types — handling all scenarios

##### 4.1 Dine-in order flow

```
Table selected (floor plan)
        ↓
Order placed (POS or QR)
        ↓
Order confirmed → KDS notified
        ↓
Items prepared (kitchen/bar in parallel)
        ↓
Items marked ready → Waiter notified
        ↓
Waiter serves items → marks delivered
        ↓
Customer requests bill
        ↓
Bill generated → Payment → Table freed
```

- Support **partial serving**: some items ready before others.
- Track **each item's serving status** independently.
- Waiter notified per item (not waiting for entire order).

##### 4.2 Takeaway order flow

```
Order placed (POS / Phone / Walk-in)
        ↓
Promised pickup time set
        ↓
Token/order number given to customer
        ↓
Kitchen prepares → Packaging station notified
        ↓
Items packed → Status: "Ready for Pickup"
        ↓
Customer arrives → Token verified
        ↓
Order handed over → Payment → Closed
```

- SMS/WhatsApp notification to customer when order is ready.
- Token display board (large screen) showing ready orders.

##### 4.3 Delivery order flow

```
Order received (app / phone / online platform)
        ↓
Order confirmed + delivery time estimated
        ↓
Kitchen prepares
        ↓
Packaging and labeling
        ↓
Delivery agent assigned
        ↓
Order dispatched → tracking started
        ↓
Delivered → Confirmation → Payment
```

#### 6. Service coordination rules engine

Implement a **rules engine** that automatically handles service scenarios:

```typescript
const coordinationRules = [

  // Rule 1: Auto-escalate slow orders
  {
    trigger: 'order_age_exceeds',
    condition: { minutes: 20, status: 'in_preparation' },
    action: 'notify_coordinator',
    message: 'Order #{order_number} taking longer than 20 min'
  },

  // Rule 2: Notify waiter when items are ready
  {
    trigger: 'item_status_changed',
    condition: { newStatus: 'ready' },
    action: 'notify_assigned_waiter',
    message: '{item_name} for Table {table_number} is ready to serve'
  },

  // Rule 3: Alert manager for VIP delays
  {
    trigger: 'vip_order_delayed',
    condition: { delayMinutes: 5 },
    action: 'notify_manager_urgent',
    message: 'VIP order #{order_number} delayed by {delay} min'
  },

  // Rule 4: Table turnover alert
  {
    trigger: 'order_paid',
    condition: { tableOccupiedMinutes: 90 },
    action: 'notify_coordinator',
    message: 'Table {table} paid after 90 min — clean and reset'
  },

  // Rule 5: Kitchen overload
  {
    trigger: 'active_kitchen_tickets_exceed',
    condition: { count: 15 },
    action: 'notify_manager',
    message: 'Kitchen overloaded: {count} active tickets'
  },

  // Rule 6: Delivery promise breach
  {
    trigger: 'delivery_time_at_risk',
    condition: { minutesToPromised: 10, status: 'in_preparation' },
    action: 'notify_coordinator_urgent',
    message: 'Delivery order #{order_number} at risk — 10 min to promised time'
  },

  // Rule 7: Unserved ready items
  {
    trigger: 'item_ready_unserved',
    condition: { waitingMinutes: 5 },
    action: 'notify_waiter_reminder',
    message: '{item_name} for Table {table} ready 5 min ago — please collect'
  },

  // Rule 8: Auto-close idle orders
  {
    trigger: 'order_status_stuck',
    condition: { status: 'served', minutesSinceLastUpdate: 30 },
    action: 'prompt_coordinator',
    message: 'Order #{order_number} served 30 min ago — request bill?'
  }
];
```
