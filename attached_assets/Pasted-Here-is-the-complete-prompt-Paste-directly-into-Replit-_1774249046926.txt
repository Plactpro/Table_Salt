### Complete Procurement & Inventory Control System

This is a continuation of the existing restaurant SaaS project in Replit. The following are already built: multi-tenant architecture, inventory management, recipe-to-inventory linking, stock deduction engine, wastage tracking, daily stock capacity report, and multi-outlet management.

Now build a **fully functional Procurement and Inventory Control System** covering the complete purchasing lifecycle — from quotation to purchase order with delivery schedules, purchase returns, inter-outlet stock transfers, physical stock verification, and damaged inventory declaration.

***

#### 1. System overview — all modules in this prompt

```
MODULE 1: SUPPLIER & QUOTATION MANAGEMENT
  → Manage suppliers, request quotations, compare prices

MODULE 2: PURCHASE ORDER (PO) MANAGEMENT
  → Generate PO directly or from approved quotation
  → Delivery schedule within PO
  → PO approval workflow
  → Goods receipt against PO

MODULE 3: PURCHASE RETURNS
  → Return damaged/wrong/excess goods to supplier
  → Debit note generation
  → Stock reversal on return

MODULE 4: INTER-OUTLET STOCK TRANSFER
  → Transfer stock between outlets/stores
  → Transfer request → approval → dispatch → receive

MODULE 5: PHYSICAL STOCK VERIFICATION
  → Scheduled and surprise stock counts
  → Variance detection and adjustment

MODULE 6: DAMAGED INVENTORY DECLARATION
  → Declare and write off damaged stock
  → Insurance claim support
  → Disposal tracking
```

      → Partially/Fully Received → Closed
