
The application must implement the following functional requirements:
1. Multi Currency Support
•	The system should support multiple currencies for all monetary values (menu prices, bills, discounts, reports).
•	Each user profile (restaurant owner, cashier, manager) can have a default currency.
•	Allow a currency selector that:
o	Automatically picks the currency from the logged in user’s profile.
o	Allows switching currency from a dropdown when permitted.
•	Implement a consistent currency formatting utility so amounts display correctly across all modules (POS, billing, reports, CRM).
•	Make it easy to plug in live exchange rates later (for now, keep rates static or configurable).
________________________________________
2. Menu & Dish Management
Create a Menu Management module for restaurant owners and managers:
•	CRUD operations for:
o	Categories (Starters, Main Course, Desserts, Beverages, etc.).
o	Individual dishes/items.
•	Each dish should support:
o	Name, description, category, base price, currency.
o	Upload and display of dish photos (store as URLs or mock file handler).
o	Tags (e.g., spicy, vegan, gluten free).
•	Customers/Waiters should be able to view:
o	Detailed course information (description, portion size, preparation style).
o	Ingredient details for each dish.
•	Implement a reusable “Ingredients Table Tab”:
o	On every major screen/tab that shows dishes, include a dedicated Ingredients / Info tab or panel.
o	When a dish is selected, this tab shows ingredient list, allergens, and nutritional notes.
o	Design it so this tab component can be reused across POS, reservations, CRM notes, etc.
________________________________________
3. Reservation Management
Create a Reservation module:
•	Allow staff (and optionally customers in future) to:
o	Create, view, edit, and cancel table reservations.
•	Capture:
o	Customer name, contact, number of guests, date/time, special requests, and assigned table.
•	Support reservation status flow:
o	Pending → Confirmed → Seated → Completed / No show.
•	Provide a calendar or time slot view for easy management of bookings.
•	Integrate reservations with:
o	Table status (free/occupied/reserved).
o	Customer profile in the CRM module.
________________________________________
4. Point of Sale (POS)
Design a robust POS module tightly integrated with menu, billing, and payment:
•	Ensure the POS screen loads correctly on the first attempt with no glitches.
•	POS tasks:
o	Select table or “takeaway/delivery” mode.
o	Add items from the menu with quantity, modifiers, and notes.
o	Apply offers/discounts (see Offers module).
•	POS should be directly connected to:
o	Billing (generate bill, tax calculation, totals).
o	Payment handling (cash, card, digital wallets; you can mock payment).
•	Optimize for fast input (keyboard shortcuts or quick buttons if feasible).
________________________________________
5. Offers & Discounts
Create an Offers & Promotions module:
•	Allow management to define promotional offers:
o	Percentage discounts, fixed amount discounts, happy hour pricing, combo deals, etc.
•	Scope options:
o	Offer at item level, category level, or entire bill.
o	Time bound offers (start and end date/time).
•	During ordering in POS:
o	Relevant offers should be clearly visible and selectable.
o	Automatically suggest applicable discounts based on cart contents.
•	Ensure discount logic is integrated into:
o	Final billing, tax calculation, and reports.
________________________________________
6. Dine In Payment Flow
Implement a proper dine in payment flow:
•	When a table is opened and items are ordered:
o	Do not show final payment prompt upfront.
•	Payment should only be enabled when:
o	Staff explicitly marks the table/order as “Ready to Pay” or “Customer Finished Dining”.
•	After this status:
o	Show bill preview, allow splitting bill (if possible), then accept payment.
•	Visually distinguish:
o	“In Progress” orders versus “Ready to Pay” versus “Paid” orders.
________________________________________
7. Delivery Module (Clarified)
Include a Delivery module placeholder with clear status:
•	Indicate that current delivery functionality is under review.
•	Provide:
o	Basic structure for delivery orders (customer address, delivery partner, order status).
•	Make it easy to:
o	Either fully disable delivery in configuration, or
o	Extend later into a fully functional delivery workflow.
•	Include comments or configuration flags to show where delivery is currently limited or inactive.
________________________________________
8. Customer Relationship Management (CRM)
Build a CRM module to manage customer interactions:
•	Customer profiles with:
o	Name, contact details, visit history, favorite dishes, average spend.
•	Loyalty features:
o	Points, tiers, or simple visit counters.
•	Feedback collection:
o	Link bills/reservations to feedback forms or rating entries.
•	Targeted promotions:
o	Ability to mark customers for campaigns (e.g., “coffee lovers”, “high spenders”).
•	Integrate CRM with:
o	Reservations, POS, and Offers (e.g., special discounts for loyal customers).
________________________________________
9. Employee Performance Tracking
Implement an Employee Performance module:
•	Track staff performance metrics such as:
o	Orders handled, upsells, average order value, shift punctuality (simplified), customer ratings.
•	Define roles: waiter, cashier, chef, manager, delivery partner.
•	Provide reports and analytics:
o	Individual employee dashboards.
o	Management view with filters (by date range, outlet, role).
•	Integrate with POS and Scheduling so performance is tied to actual shifts and orders.
________________________________________
10. Billing Management
Create a comprehensive Billing module:
•	Handle:
o	Invoices, receipts, tax, service charges, and payment records.
•	Integrate with:
o	POS (bill creation and closing).
o	Multi currency support (display currency, convert if needed).
•	Store and show:
o	Bill history per customer, per table, per day/outlet.
•	Structure the code so it can later be connected to external accounting systems (e.g., via API).
________________________________________
11. Staff Scheduling
Build a Staff Scheduling module:
•	Create and manage shift schedules:
o	Assign employees to specific shifts (date, time, role, outlet).
•	Allow managers to:
o	Add, edit, and delete shifts.
o	Mark attendance or basic status (present/absent).
•	Provide a simple weekly/monthly view of staffing.
•	Connect scheduling to:
o	Employee Performance module (to see performance per shift).
o	CRM and POS (know who handled which orders).
________________________________________
Cross Cutting Technical & UX Requirements
•	Use a modular architecture: separate components for POS, Reservations, CRM, Billing, etc.
•	Centralize:
o	User profile & role management (manager, cashier, waiter, admin).
o	Currency configuration and formatting.
•	Provide consistent navigation with a left side menu and a clean, rich theme.
•	Ensure the UI is:
o	Responsive (desktop and tablet at minimum).
o	Intuitive for restaurant staff with minimal training.
•	Include sample/demo data so each module is easily testable.
Goal:
Deliver a clear, well structured restaurant management application prototype that demonstrates all the above modules and flows working together, with clean code and clear separation of concerns so it can be extended into a full production system later.
