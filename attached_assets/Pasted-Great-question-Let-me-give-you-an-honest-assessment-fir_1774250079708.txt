## Is This a Good Approach?

**Yes — absolutely.** This is called **Selective Item Cooking Control** and it solves real kitchen problems.

### Why This Is Needed

```
REAL SCENARIO:
Order #047 — Table 5:
  1. Caesar Salad      → Ready in 5 min (Cold Kitchen)
  2. Grilled Chicken   → Ready in 15 min (Grill)
  3. Chocolate Cake    → Ready in 3 min (Dessert)
  4. Soup of the Day   → Ready in 8 min (Hot Counter)

❌ WITHOUT selective start:
   All 4 items start cooking simultaneously
   Caesar Salad ready at 0:05
   Cake ready at 0:03
   Both sitting cold while waiting for Chicken (0:15)
   Customer gets cold salad and melted cake

✅ WITH selective start:
   Chef starts Grilled Chicken immediately (15 min)
   At 7 min → starts Soup (8 min)
   At 10 min → starts Caesar Salad (5 min)
   At 12 min → starts Chocolate Cake (3 min)
   ALL 4 items ready together at 15 min ← PERFECT
```

### 3 Scenarios Where This Is Critical

| Scenario | Problem Solved |
|---|---|
| **Mixed prep time items** | Fast items wait for slow ones — served together hot |
| **Course-wise serving** | Starters cook first, mains later, desserts last |
| **Station coordination** | Bar starts drinks only when food is almost ready |
| **Rush hour management** | Chef prioritizes urgent tables, defers others |

### Selective Item Cooking Control in KDS

This is a continuation of the existing restaurant SaaS project in Replit. The following are already built: multi-tenant architecture, KDS wall screen and chef tablet, chef assignment system, order management, and food modifications.

Now build a **Selective Item Cooking Control system** in the KDS that allows chefs to start cooking specific items individually — with intelligent timing suggestions, course-based sequencing, and coordinator visibility — instead of being forced to start all items in an order simultaneously.

#### 1. Core concept

```
ORDER ARRIVES ON KDS
        ↓
Items shown individually — NOT all auto-started
        ↓
Each item has its own START button
        ↓
Chef decides WHICH item to start and WHEN
        ↓
System provides SMART TIMING SUGGESTIONS

        ↓
Items tracked independently:
  → Own status
  → Own timer
  → Own chef assignment
        ↓
Order marked FULLY READY only when
ALL items are in READY status
```

#### 3. Smart timing suggestion engine

```typescript
interface TimingContext {
  orderId: string;
  targetReadyTime?: Date;
  // if null: calculate based on longest item
}

async function calculateSuggestedStartTimes(
  orderItems: OrderItem[],
  context: TimingContext
): Promise<ItemTiming[]> {

  // Step 1: Get prep time for each item from recipe
  const itemsWithPrepTime = await Promise.all(
    orderItems.map(async item => ({
      ...item,
      prepTime: await getRecipePrepTime(item.menuItemId)
        ?? item.estimatedPrepMinutes
        ?? 15 // fallback default
    }))
  );

  // Step 2: Find the longest prep time item
  const maxPrepTime = Math.max(
    ...itemsWithPrepTime.map(i => i.prepTime)
  );

  // Step 3: Target ready time = now + longest prep time
  const targetReady = context.targetReadyTime
    ?? addMinutes(new Date(), maxPrepTime);

  // Step 4: Calculate start time for each item
  return itemsWithPrepTime.map(item => ({
    itemId: item.id,
    itemName: item.menuItemName,
    prepTimeMinutes: item.prepTime,
    suggestedStartAt: subMinutes(targetReady, item.prepTime),
    estimatedReadyAt: targetReady,
    startIn: differenceInMinutes(
      subMinutes(targetReady, item.prepTime),
      new Date()
    )
    // positive = start in X min, negative = start NOW/overdue
  }));
}
```

#### 5. Item action buttons and behaviors

##### ▶ START button
- Chef taps START on a specific item.
- Status: `queued` → `started`.
- Timer begins counting down from prep time.
- If started BEFORE suggested time:
  - Show subtle info: "Started 5 min early — item may be ready before others"
- If started AFTER suggested time:
  - Show amber warning: "⚠️ Started late — this item may delay the order"

##### ⏸️ HOLD button
- Chef deliberately delays an item.
- Popup appears:
  ```
  ┌────────────────────────────────┐
  │  HOLD ITEM                     │
  │  Caesar Salad — Table 5        │
  │                                │
  │  Hold until:                   │
  │  ○ I manually start it         │
  │  ○ Grilled Chicken is ready    │
  │  ○ In  minutes             │
  │                                │
  │  Reason (optional):            │
  │  [Waiting for grill station ]  │
  │  [HOLD ITEM]                   │
  └────────────────────────────────┘
  ```
- Status: `hold`
- When hold condition is met: auto-reminder to chef

##### ✅ READY button
- Chef marks specific item as done.
- Status: `started` → `ready`
- Waiter notified: "Caesar Salad for Table 5 is ready"
- If all items ready: order status → `fully_ready`
- If some items still cooking: order status → `partially_ready`

##### 🔥 RUSH button (override)
- Start all remaining items immediately.
- For urgent situations (VIP, long wait complaint).
- Requires manager PIN or tap to confirm.
- All items started, timing warnings shown.
