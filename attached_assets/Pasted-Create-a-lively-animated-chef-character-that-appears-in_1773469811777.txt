Create a **lively animated chef character** that appears in the **bottom-left corner** of a web page (for a restaurant management app). The character should be engaging, friendly, and react to user interactions to make the interface feel more alive and fun.

## **Character Design Requirements:**

**Appearance:**
- **Friendly chef mascot**: Middle-aged chef with a big smile, chef hat (tall toque), white chef jacket with rolled sleeves, apron, and a colorful neckerchief.
- **Cartoonish style**: Expressive face with big eyes, rosy cheeks, and animated eyebrows. Slightly chubby and approachable (like Pixar character).
- **Size**: 200-250px height when idle, scalable to 300px for reactions.
- **Colors**: Vibrant but not overwhelming – white uniform, red/white checkered apron, colorful hat band, warm skin tones.

**Positioning:**
```
Position: fixed; bottom: 20px; left: 20px; z-index: 999;
Responsive: Scale down on mobile (150px), move up slightly if keyboard appears.
```

## **Animation States & Reactions (Lottie JSON preferred, or pure CSS/JS)**

### **1. Idle State (Looping)**
- Gentle breathing + subtle sway side-to-side.
- Occasional actions: Wipe forehead with handkerchief, adjust hat, tap foot rhythmically.
- Duration: 4-6 second loop.

### **2. Greeting/Welcome (On page load)**
- Wave enthusiastically with one hand while holding a spatula in the other.
- Big smile, eyes sparkle, slight hop/jump.
- Auto‑play once on load, then return to idle.

### **3. User Interaction Reactions**

**A. Mouse hover near character:**
```
Happy wave + speech bubble ("Hi Chef!") → fade out.
Eyes follow cursor briefly, then back to center.
```

**B. Page scroll (detect scroll events):**
```
Chef "looks up" (head tilt), gives thumbs up, returns to idle.
```

**C. Click/tap on character:**
```
Bows politely or does a happy spin.
Speech bubble: "Ready to cook!" → fade out.
```

**D. Form success (listen for specific events):**
```
Chef claps hands excitedly, jumps up and down.
Confetti or steam/heart particles around head.
```

**E. Error/warning (custom event):**
```
Chef looks surprised/worried (raised eyebrows), shrugs shoulders.
Quick recovery to encouraging thumbs up.
```

**F. Idle timeout (5+ min no activity):**
```
Chef yawns, stretches arms, pretends to nap, then perks up when user moves mouse.
```

### **4. Contextual Reactions (Restaurant App Specific)**
```
- POS order completed: Chef tastes food, gives "Perfect!" gesture.
- New reservation: Chef sets table (mime action with invisible plates).
- Busy hour detected: Chef looks busy, juggling pans quickly.
```

## **Technical Implementation:**

**Option 1: Lottie Animation (Recommended)**
```
- Use Bodymovin/After Effects exported Lottie JSON.
- Include multiple animation segments triggered by JS events.
- Library: lottie-web or lottie-player.
- CSS classes for color customization (.chef-hat, .chef-apron).
```

**Option 2: Pure CSS + SVG (Fallback)**
```
- SVG sprite with multiple poses/frames.
- CSS @keyframes for each animation state.
- Intersection Observer + mouse events for triggers.
```

**Option 3: CSS + Canvas (Advanced)**
```
- Canvas-based character with particle effects.
- Smooth interpolation between poses.
```

**Sample Trigger Code:**
```javascript
const chef = document.getElementById('chef-container');

// Mouse enter
chef.addEventListener('mouseenter', () => playAnimation('wave'));

// Scroll
window.addEventListener('scroll', () => {
  if (window.scrollY > 100) playAnimation('thumbs-up');
});

// Custom events
document.addEventListener('order-completed', () => playAnimation('taste-food'));
```

## **Interaction Priority & Smoothness:**
```
1. Never block user interactions (pointer-events: none when appropriate).
2. Smooth transitions between states (0.3s ease).
3. Reduce motion option (prefers-reduced-motion media query).
4. Performance: 60fps, lightweight (<100KB total).
```

## **Deliverables:**
1. **Complete HTML/CSS/JS code** for the animated chef (self-contained).

3. **Demo page** showing all reactions working.
4. **Integration guide** for embedding into existing restaurant app.
5. **Customization variables** (colors, size, reactions) in CSS custom properties.

Make restaurant staff smile every time they use the app. The chef should feel like a helpful kitchen companion who celebrates successes and encourages during busy times!
