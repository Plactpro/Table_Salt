## My Suggestions Before Running This Prompt

### Suggestion 1 — Add Data Retention Configuration UI

```
The schema has retentionPolicyMonths
but there is no UI for owner to set it.

  Data Retention Policy: [36 months ▼]
  Auto-delete after retention period: [✅]

This makes the compliance report
more meaningful and configurable.
```

### Suggestion 2 — Add CERT-In Notification Tracking

```
CERT-In is India's Computer Emergency
Response Team — mandatory notification
within 6 hours for certain breach types
(stricter than GDPR's 72 hours).

Add to breach_incidents table:
  certin_notified     BOOLEAN DEFAULT false,
  certin_notified_at  TIMESTAMP,
  certin_reference_no VARCHAR(100)

This costs you 3 extra columns but
covers Indian regulatory requirement.
```

### Suggestion 3 — Version the Compliance Report

```
Currently the report generates live data.

Add:
  → Save snapshots when downloaded
  → Store in compliance_report_snapshots table
  → Owner can compare current vs 3 months ago
  → Auditors love historical snapshots

This is a Phase 2 addition but worth planning.
```

### Suggestion 4 — Add Breach Severity Clarification

```
Your severity levels are:
  low / medium / high / critical

For GDPR Article 33 specifically:
  Not all breaches require DPA notification
  Only those "likely to result in risk to
  rights and freedoms of natural persons"

Add a field:
  requires_dpa_notification BOOLEAN DEFAULT false
  notification_rationale    TEXT

This prevents over-reporting to authorities
for low-risk incidents.
```
