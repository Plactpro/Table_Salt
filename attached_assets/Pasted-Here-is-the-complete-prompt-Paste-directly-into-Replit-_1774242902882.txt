### Food Wastage Tracking System

This is a continuation of the existing restaurant SaaS project in Replit. The following are already built: multi-tenant architecture, recipe-to-inventory linking, stock deduction engine, chef assignment, pre-prep task system, KDS, shift reconciliation, and daily stock capacity report.

Now build a **fully functional Food Wastage Tracking System** that captures every type of food waste across the entire restaurant operation — kitchen prep waste, cooking errors, expired stock, over-production, plate returns, and spoilage — with real-time logging, cost impact calculation, staff accountability, and actionable reports to reduce waste over time.

***

#### 1. Wastage types — cover all scenarios

```sql
CREATE TYPE wastage_category AS ENUM (
  'PREP_WASTE',        -- trimming, peeling, chopping offcuts
  'COOKING_ERROR',     -- burnt, overcooked, wrong recipe
  'OVER_PRODUCTION',   -- made too much, unsold at end of shift
  'PLATE_RETURN',      -- customer returned dish
  'SPOILAGE',          -- expired or spoiled before use
  'STORAGE_DAMAGE',    -- damaged in storage (dropped, spilled)
  'QUALITY_REJECTION', -- failed quality check before serving
  'TRANSFER_LOSS',     -- lost during portioning or transfer
  'TRIAL_BATCH',       -- test cooking for new dishes
  'STAFF_MEAL',        -- food consumed by staff (tracked separately)
  'BUFFET_LEFTOVER',   -- unsold buffet items at end of service
  'EVENT_LEFTOVER',    -- leftover from catering/events
  'OTHER'              -- custom reason with mandatory note
);
```

#### 2. Database schema

```sql
-- Main wastage log
CREATE TABLE wastage_logs (
  id                  UUID PRIMARY KEY,
  tenant_id           UUID NOT NULL,
  outlet_id           UUID NOT NULL,
  wastage_number      VARCHAR(50) UNIQUE NOT NULL,
    -- format: WST-20260323-0001
  wastage_category    wastage_category NOT NULL,
  wastage_stage       VARCHAR(30) NOT NULL,
    -- 'RAW_INGREDIENT', 'SEMI_PREPARED', 'FULLY_PREPARED',
    -- 'PLATED', 'RETURNED_BY_CUSTOMER'
  ingredient_id       UUID,
  ingredient_name     VARCHAR(255),
  menu_item_id        UUID,
  menu_item_name      VARCHAR(255),
  order_id            UUID,
  order_number        VARCHAR(50),
  quantity_wasted     DECIMAL(10,3) NOT NULL,
  unit                VARCHAR(20) NOT NULL,
  unit_cost           DECIMAL(10,2),
  total_cost          DECIMAL(10,2),
  currency            VARCHAR(10) DEFAULT 'INR',
  reason              TEXT NOT NULL,
  preventable         BOOLEAN,
    -- was this waste preventable?
  prevention_note     TEXT,
  logged_by           UUID NOT NULL,
  logged_by_name      VARCHAR(255),
  logged_by_role      VARCHAR(50),
  counter_id          UUID,
  counter_name        VARCHAR(100),
  chef_id             UUID,
  chef_name           VARCHAR(255),
  shift_id            UUID,
  shift_type          VARCHAR(20),
  image_url           TEXT,
    -- optional photo of waste for audit
  is_recovered        BOOLEAN DEFAULT false,
    -- e.g. vegetable trim used for stock
  recovery_note       TEXT,
  approved_by         UUID,
  approved_at         TIMESTAMP,
  wastage_date        DATE NOT NULL,
  wastage_time        TIME NOT NULL,
  created_at          TIMESTAMP DEFAULT NOW(),
  updated_at          TIMESTAMP DEFAULT NOW()
);

-- Daily wastage summary per outlet
CREATE TABLE wastage_daily_summary (
  id                  UUID PRIMARY KEY,
  tenant_id           UUID NOT NULL,
  outlet_id           UUID NOT NULL,
  summary_date        DATE NOT NULL,
  shift_type          VARCHAR(20),
  total_entries       INT DEFAULT 0,
  total_cost          DECIMAL(10,2) DEFAULT 0,
  preventable_cost    DECIMAL(10,2) DEFAULT 0,
  recovered_cost      DECIMAL(10,2) DEFAULT 0,
  net_loss            DECIMAL(10,2) DEFAULT 0,
  by_category         JSONB,
    -- breakdown of cost per category
  by_counter          JSONB,
    -- breakdown of cost per counter
  by_chef             JSONB,
    -- breakdown of cost per chef
  top_wasted_item     VARCHAR(255),
  created_at          TIMESTAMP DEFAULT NOW(),
  UNIQUE(tenant_id, outlet_id, summary_date, shift_type)
);

-- Wastage targets per outlet (budget for acceptable waste)
CREATE TABLE wastage_targets (
  id                  UUID PRIMARY KEY,
  tenant_id           UUID NOT NULL,
  outlet_id           UUID NOT NULL,
  target_type         VARCHAR(20) DEFAULT 'daily',
    -- 'daily', 'weekly', 'monthly'
  target_amount       DECIMAL(10,2) NOT NULL,
    -- max acceptable wastage cost
  target_percentage   DECIMAL(5,2),
    -- max % of daily revenue acceptable as waste
  alert_threshold     DECIMAL(5,2) DEFAULT 80,
    -- alert when X% of target is reached
  effective_from      DATE NOT NULL,
  created_by          UUID,
  created_at          TIMESTAMP DEFAULT NOW()
);
```

#### 4. Recovery tracking

Not all waste is a total loss. Track recovered waste:

```
RECOVERY TYPES:
  ├── Used for staff meal     → cost moved to staff meal budget
  ├── Used for stock making   → vegetable trim → soup stock
  ├── Sold at discount        → near-expiry items at reduced price
  ├── Donated                 → food bank / charity
  └── Composted               → environmental credit

For each wastage entry:
  ├── is_recovered: true/false
  ├── recovery_type: (above options)
  ├── recovery_value: ₹ amount recovered
  └── Net loss = Total cost - Recovery value
```
