### Prompt for Replit AI – Enhanced POS + Billing + Payments Module

You are building the **complete Point of Sale (POS), Billing, and Payment module** for a multi-tenant restaurant SaaS. This module is the core revenue engine of the application. It must connect seamlessly with the already-built **Kitchen Display System (KDS), Recipe-Inventory engine, Menu management, Table management, and Customer CRM**.

***

Build a **single-page, split-panel POS interface** optimized for both desktop and tablet:

- Left panel: menu browser with search, categories, item cards, and combo section.

***

#### 2. Order creation flow

##### Step 1 – Start new order
- Waiter opens POS → selects order type:
  - **Dine-In**: select table from visual floor plan mini-view or table number dropdown.
  - **Takeaway**: enter customer name and phone (optional).
  - **Delivery**: enter customer name, phone, and delivery address (or select existing customer from CRM).
- Support **multiple open orders simultaneously** per waiter session.
- Show open orders as tabs at the top of the cart panel.

##### Step 2 – Add items to cart
- Left panel displays:
  - Search bar (search by name, tag, or SKU).
  - Category tabs (horizontal scroll): All / Starters / Mains / Desserts / Drinks / Combos.
  - Item cards showing:
    - Item image (thumbnail)
    - Name and description (1 line)
    - Price
    - Veg/non-veg badge
    - Availability indicator
    - Quick **"+" button** to add to cart.
  - **Combo Offers section** (highlighted with savings badge):
    - Show combo name, combo price, strikethrough individual price, and % savings badge.
    - Tapping a combo adds it as a single line item.

- When item is added to cart:
  - Small bounce animation on cart icon.
  - Item appears in right panel cart with quantity controls (+ / −).

##### Step 3 – Item customization
- Tapping an item in the cart opens a **modifier drawer**:
  - Show available modifiers (size, extras, toppings) with prices.
  - Special instructions text field (free text, sent to kitchen).
  - Quantity adjuster.
- Modifier selections update line item price in real time.

##### Step 4 – Cart management
- Each cart line shows:
  - Item name + modifiers
  - Quantity (tap to edit)
  - Line total
  - Delete (swipe left or trash icon)
- Support **splitting the order** into multiple sub-orders:
  - Split by number of guests (divide items equally or manually assign).
  - Split by selecting specific items per guest.
  - Each split generates a separate bill.
- **Hold order**: save current cart without sending to kitchen (for later).
- **Recall order**: retrieve a held order.

***

#### 3. Discounts, charges, and taxes

All applied in real time in the cart:

- **Discount**:
  - Flat amount (e.g., −₹50) or percentage (e.g., 10%).
  - Apply to entire order or specific line items.
  - Require reason entry (dropdown: loyalty redemption / manager override / promotional / other).
  - Role-based: Waiter can apply up to 10%; Manager can apply up to 30%; Owner unlimited.
- **Service charge**:
  - Configurable per outlet (e.g., 5%), auto-applied.
  - Toggle on/off per order if allowed.
- **Tax**:
  - Support multiple tax rules (GST, VAT, etc.) configured in Settings.
  - Auto-apply based on item tax group.
  - Show tax breakdown in cart (tax name + amount).
- **Tips**:
  - Quick-select buttons: 5% / 10% / 15% / Custom amount.
  - Tips excluded from tax calculation.
- **Loyalty points redemption**:
  - If customer is linked to order, show available loyalty points.
  - Allow redemption: points convert to discount amount.

***

#### 4. Send to kitchen (auto-routing)

- **"Send to Kitchen"** button in cart panel:
  - Sends all un-sent items to KDS.
  - Auto-splits KOT by station (already built in Prompt 2).
  - Items marked as "sent" in cart (greyed out, cannot be edited without void).
  - New items added after initial send are highlighted as **"Add-on"** in KDS.
- If outlet display mode is `WALL_SCREEN` or `BOTH`:
  - Ticket appears on wall screen instantly via WebSocket.
- If display mode is `TABLET`:
  - Ticket appears on relevant chef tablets by station.
- Log KOT event in `kot_events` table with timestamp and waiter ID.

##### Bill database schema
```sql
CREATE TABLE bills (
  id                UUID PRIMARY KEY,
  tenant_id         UUID NOT NULL,
  outlet_id         UUID NOT NULL,
  bill_number       VARCHAR(50) UNIQUE NOT NULL,
  order_id          UUID NOT NULL,
  table_id          UUID,
  customer_id       UUID,
  waiter_id         UUID NOT NULL,
  waiter_name       VARCHAR(255),
  subtotal          DECIMAL(10,2),
  discount_amount   DECIMAL(10,2) DEFAULT 0,
  discount_reason   VARCHAR(255),
  service_charge    DECIMAL(10,2) DEFAULT 0,
  tax_amount        DECIMAL(10,2) DEFAULT 0,
  tax_breakdown     JSONB,
  tips              DECIMAL(10,2) DEFAULT 0,
  total_amount      DECIMAL(10,2) NOT NULL,
  payment_status    VARCHAR(20) DEFAULT 'pending',
    -- pending, partially_paid, paid, voided
  created_at        TIMESTAMP DEFAULT NOW(),
  paid_at           TIMESTAMP
);

CREATE TABLE bill_payments (
  id             UUID PRIMARY KEY,
  tenant_id      UUID NOT NULL,
  bill_id        UUID NOT NULL,
  payment_method VARCHAR(30) NOT NULL,
    -- 'CASH', 'CARD', 'UPI', 'WALLET', 'LOYALTY'
  amount         DECIMAL(10,2) NOT NULL,
  reference_no   VARCHAR(100),
  collected_by   UUID,
  created_at     TIMESTAMP DEFAULT NOW()
);
```

#### 6. Payment processing

After bill preview, show a **Payment Screen**:

- Display total amount due prominently.
- Payment method selection (multi-payment supported):
  ```
  [💵 Cash]  [💳 Card]  [📱 UPI/QR]  [🎁 Loyalty]  [+ Split Payment]
  ```

- **Cash payment**:
  - Enter amount tendered.
  - Auto-calculate and display change due.
  - Example: Total ₹992 → Tendered ₹1000 → Change ₹8.

- **Card payment**:
  - Enter last 4 digits and reference/approval code.
  - Integration placeholder for Stripe / Razorpay terminal.

- **UPI / QR**:
  - Generate dynamic QR code for the bill amount (Razorpay / UPI deep link placeholder).
  - Show "Waiting for payment confirmation" with auto-refresh.
  - Manual confirmation button as fallback.

- **Loyalty points**:
  - If customer linked: show available points and equivalent discount.
  - Deduct points on payment confirmation.
  - Log loyalty transaction.

- **Split payment** (multiple methods for one bill):
  - Add payment rows: each row has method + amount.
  - Running balance shows how much is still due.
  - Example: ₹500 by card + ₹492.25 by cash = ₹992.25 settled.

- On payment completion:
  - Bill status → `paid`.
  - Table status → `free` (for dine-in).
  - Loyalty points earned → added to customer profile.
  - Order status → `completed`.
  - Stock deduction triggered (if not already done at preparation stage).
  - Receipt generated and available for print/download/email.

***

#### 7. Receipt output options

After payment, show action buttons:

- **Print Receipt**: send to connected thermal printer (use browser print API with receipt CSS).
- **Download PDF**: generate downloadable PDF of the bill.
- **Email Receipt**: send to customer email (if provided).
- **WhatsApp Receipt**: generate WhatsApp share link with bill summary (optional).
- **New Order**: immediately start a new POS session for the next table.

***

#### 8. Void and refund flow

- **Void bill** (before payment):
  - Manager-only action with mandatory reason.
  - Triggers stock reversal via `RECIPE_REVERSAL`.
  - Table freed, order marked voided.

- **Refund** (after payment):
  - Full or partial refund.
  - Reason required.
  - Refund amount logged in `bill_payments` with negative amount.
  - Loyalty points reversed if redeemed.
  - Inventory NOT reversed automatically (wastage assumed); manager can manually adjust.

***

#### 9. POS session and shift management

- Each waiter starts a **POS session** (shift open):
  - Record opening cash float amount.
  - All orders and payments tied to this session.
- Session close (shift end):
  - Show session summary:
    - Total orders, total revenue, breakdown by payment method.
    - Cash expected (float + cash sales) vs cash counted.
    - Cash variance highlighted.
  - Manager must approve session close.
  - Session report exportable as PDF.

#### 10. POS performance and UX requirements

- Item search must return results within **200ms** (use indexed DB queries).
- Cart calculations must update in **real time** as items are added/modified.
- All buttons touch-friendly: minimum **48px tap target**.
- Offline resilience:
  - Cache menu items and open orders locally (IndexedDB or localStorage).
  - Queue orders locally if connection drops; sync when reconnected.
  - Show a "Offline Mode" banner when disconnected.
- Keyboard shortcuts for desktop POS:
  - `Ctrl+N` = new order
  - `Ctrl+B` = open bill
  - `Ctrl+K` = send to kitchen
  - `Escape` = close drawer/modal

#### 11. APIs required

```
POST   /api/orders                    → create new order
PUT    /api/orders/:id/items          → add/update items
POST   /api/orders/:id/send-kitchen   → send to KDS
GET    /api/orders/:id/bill           → generate bill preview
POST   /api/bills                     → create bill
POST   /api/bills/:id/payments        → add payment
PUT    /api/bills/:id/void            → void bill
POST   /api/bills/:id/refund          → process refund
GET    /api/pos/session               → get current session
POST   /api/pos/session/open          → open session
POST   /api/pos/session/close         → close session
GET    /api/pos/session/report        → session summary
```

#### 12. Integration points (connect to existing modules)

- **KDS**: `send-kitchen` API triggers WebSocket broadcast to wall screen and chef tablets.
- **Inventory**: payment completion triggers `applyRecipeConsumption(orderId)` if not already triggered at preparation.
- **Menu**: left panel reads from existing menu/combo/modifier APIs.
- **Tables**: order creation updates table status to `occupied`; payment completion frees the table.
- **Customer CRM**: link customer to order; update visit history and loyalty points on payment.
- **Reports**: all bills and payments feed into sales reports and finance dashboard.
